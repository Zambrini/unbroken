# UNBROKEN Direction Rules

## Promise

UNBROKEN is a fixed-camera 2D neon boss arena. One globally watched challenger
reads fair deterministic attacks, uses a responsive dash and precise shots to
punish HEIR, faces one explicit bounded mutation after victory, and becomes the
last recorded conqueror.

## Director Loop

1. Play the latest shipped build before choosing work.
2. Select the single highest-leverage player-visible weakness.
3. Define one bounded checkpoint with rendered-game acceptance criteria.
4. Implement without adding parallel systems.
5. Verify tests, build, natural gameplay, rendered states, console/network, and
   the public deployment in proportion to risk.
6. Update `VERSION`, `CHANGELOG.md`, `ROADMAP.md`, `DEVLOG.md`, QA evidence, and
   the companion website.
7. Commit, tag, push, deploy, smoke-test, then begin the next cycle.

When visuals are the limiting weakness, compare the live build with the relevant
image in `docs/concepts/` before choosing the checkpoint.

## Preserve

- Humanity's-champion fantasy and live-event framing.
- Fixed top-down 16:9 camera.
- Cyan player/action, dashed magenta forecast, solid red danger, white opening.
- Desktop keyboard/mouse gameplay and mobile read-only spectating.
- Deterministic authored attacks and bounded between-victory mutation.
- Telegraph fairness, one active question, and a reachable safe answer.

## No-Go Without Daniel

Do not change the core promise or add accounts, chat, cosmetics, inventory,
metagame progression, multiple modes or bosses, controller/touch gameplay,
opaque random difficulty, frame-by-frame AI control, paid infrastructure,
secrets, or production-scale networking without explicit approval.
